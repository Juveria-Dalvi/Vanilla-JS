GIT

Git is the free and open source distributed version control stytem that's responsible for everything Github related that happens locally on your computer. Working with Git on the command line can be challenging for absolute begginer. To help with that, Iâ€™ve put together a list of common Git commands to get started with, and which is used all the time!!

opening git for first time!! Don't worry it's not that hard, all you have to do is setup your email-ID and username

Go to the folder > right click to open 'git bash here', if the user is new then setup name and email :
  • $ git-config   --global user.name bella hadid
  • $ git-config   --global user.email bellahadid@gmail.com
This will show your data saved (which will contain your name and email) :
  • $ git-config   --list
Also you can check it by using these command lines :
  • $ git-config   user.name
  • $ git-config   user.email
After configuring your email and name, run this command to check if the repository is already contains .git (which will be a hidden file in your folder) :
  • $ git   status
If the above command gives error and says 'fatal' not a git repository, then run below command which will create empty git repository with hidden file which will track changes:
  • $ git   init
check the status after initialization :
  • $ git   status
Stage all the files using these command, OR use dot to stage all files :
  • $ git   add   --a
  • $ git   add   .
Use git status to check the file status, it will show some changes in green color if you stage file :
  • $ git   status
Commit the files with THE "commit message" :
  • $ git   commit -m "removed a bug"
Use git status to check the file status, it will show; On branch master nothing to commit, working tree clean :
  • $ git   status
To see all commits use this command:
  • $ git   log
If you want to remove your tracked changes and delete .git hidden repository use below command [NOTE:it will permanently remove tracked changes and delete .git ] :
  • $ rm   -rf.git
